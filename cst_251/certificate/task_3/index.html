Introducing HTML

HTML stands for HyperText Markup Language. It is commonly used as the baseline and foundation for many web pages and is a good starting place for those looking into designing their own websites. HTML is just a file, and thus, can be edited or created using any plain text editor. Some examples of text editors include:

  • Sublime3
  • NotePad++
  • TextEdit
  • Atom
  • Vim

The primary purpose of HTML is to provide content and structure to web pages. These elements can then be customized and enhanced with either CSS or JS.

Boilerplate

In order to provide consistency, as well as providing an ease of use for developers, a boilerplate has been established for HTML. The boilerplate acts as a template from which many HTML pages are based upon, and this similar code is commonly shared. It offers a stable starting point from which developers can build upon and is a great foundation for web pages.

A common iteration of the HTML Boilerplate is as follows:

<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<meta name="viewport" content="width=device-width">
<title></title>
</head>
<body>

</body>
</html>

For best practice, it's also recommended to include a lang attribute within the <html> tag. For example, a website which is in english should include the lang="en" attribute with the <html> tag. This feature assists search engines in finding websites that are the same language as a user's machine.

Anatomy of Elements

Within HTML files there are various different HTML elements that make up the website. These elements introduce content to the web page and tell the browser what type of content should be displayed and where. Most HTML elements follow an anatomy similar to this example:

<nameoftag attribute="some value">
Content
</nameoftag>

It's important to note that not all elements follow this convention. Most notably, there are quite a few elements where a closing tag is not needed. These elements include:

<img>
<br>
<hr>

The tag <img> is used to insert images and does not require a closing tag. It's very important to note, however, that best practice entails a <figure> tag, which does come with a corresponding closing tag: </figure>. The tag <br> introduces a break line in between text, and the <hr> tag implements a horizontal line across the web page.

Common Elements

Elements in HTML primarily see variety in the aspect of which tags are used. There are many, many tags that are utilized in HTML, so this section will only be covering some of the more common tags to help beginners.

One of the most common category of tags are the heading tags which can range from <h1></h1> to <h6></h6>, where the size of the heading gets smaller as the number increases. These tags are mainly used for titles or subheadings.

In order to add images, the <img> tag should be added. This tag should also include the src="" attribute, which will serve as a pathway, letting the server know which image belongs in this <img> tag. For the best results, the <img> tag should also include the alt="" attribute, like so:

<img src="img/.jpg" alt="image">

To add a link, the <a> tag should be used. This tag will require the href="" attribute, which is where the actual link will be placed. An example is as follows:

<a href="https://www.wikipedia.org/" target="_blank"> Wikipedia <a>

List can also be added by using HTML and they come in the form of ordered list by number (<ol></ol>), and unordered list using bullet points(<ul></ul>). Within these tags, the <li> tag should be used to identify the list items.

The following example demonstrates how to properly set up an ordered list:

<ol>
<li>First</li>
<li>Second</li>
<li>Third</li>
<ol>

Nesting Elements

An important aspect of writing and producing HTML files is organization. Just like how HTML provides structure to the web page, the code of HTML documents must also have good structure that allows development to be more streamlined, whoever the developer may be. One of the ways in which HTML is organized is through the nesting of elements. Nesting an element means placing an element inside another element, and indenting to indicate that the element is nested. This is commonly done with the <div> tag, as seen in this example:

<div>
<h1> Picture </h1>
<figure id="image-box">
<img src="img/.jpg" alt="image">
<figcaption>
A very cool caption
</figcaption>
</figure>
</div>

The further a tag is within the nested element, the more elements it's inside of. It can somewhat be interpreted similarly to the concept of a Russian Matryoshka Doll.

User Input Elements

Sometimes for certain sites, gathering information or data from people who visit the site could be useful. One way of achieving this is through the use of form elements where users can answer questions in a survey and then promptly, submit their answers. All of these different form tags should be embedded within a <form></form> tag.

Some common form tags include:

<input type="text" name="something">
<input type="email" name="something">
<input type="number" name="something">
<textarea name="something"></textarea>
<input type="radio" name="something">
<input type="checkbox" name="something">

These elements can be further customized with additional attributes that allow for the collection of more specific data.
